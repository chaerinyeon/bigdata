# 스타트업 생존율 분석 및 최적 투자 포트폴리오 구축

*Spark 분산 처리와 몬테카를로 시뮬레이션 기반 벤처 투자 포트폴리오 분석 | 빅데이터분석 기말 프로젝트 1조*

---

## 1. 문제 정의

벤처 투자자는 한정된 자본을 여러 산업·국가의 스타트업에 배분할 때 **"폐업 가능성이 낮은 곳임과 동시에 성공(인수·IPO) 가능성은 높은 기업"** 을 판단해 투자합니다. 그러나 공개 데이터에는 펀드 수익률(IRR)이 없으므로, 본 분석은 성과를 다음과 같이 설정했습니다.

- **폐업률:** `status == closed` 비율 (낮을수록 안정적)
- **성공률:** `status ∈ {acquired, ipo}` 비율 (성공 엑시트)
- **생존율:** `status != closed` 비율

이 기준을 바탕으로 국가·산업 단위의 위험을 측정하고, 이를 입력으로 몬테카를로 시뮬레이션을 실행하여 **위험 대비 기대 성공률이 가장 높은 자본 배분**을 탐색합니다.

- **해결 주체:** 초기 단계 벤처 투자자 / 액셀러레이터
- **분석 목표:** ① 국가·산업별 폐업/성공률 정량화 ② 위험 대비 수익이 효율적인 자본 배분안 도출

## 2. 선행자료 조사 및 포지셔닝

스타트업 투자 의사결정은 이론모델을 다루는 학계와 실증통계를 다루는 산업 양쪽에서 다뤄져 왔습니다. 대표 선행연구 2건을 검토해 본 프로젝트의 위치를 설정합니다.

| 선행 연구 | 핵심 내용 | 한계 (미해결 과제) | 본 프로젝트의 차별점 |
|-----------|-----------|--------------------|----------------------|
| **Ferreira & Pereira (2021), _EJOR_** — VC 진입·엑시트 투자결정 동적 모델 | 실물옵션 기반으로 진입·엑시트 **타이밍**, 최적 지분, 기대 cash multiple(≈10.2)을 도출 | 가정(성장률·시너지·변동성) 기반 **이론·해석 모델**로, 실제 기업 데이터로 검증되지 않음 | 동일한 "투자 의사결정" 문제를 **실데이터 + 시뮬레이션**으로 실증 |
| **CB Insights — The Venture Capital Funnel** | 시드 투자 코호트를 추적해 단계별로 **극소수만 후속 라운드·엑시트에 도달**하는 "깔때기" 구조를 제시 | 산업 전체의 **기술 통계(descriptive)** 로, 산업·국가별 차이나 자본 배분 처방은 없음 | 국가·산업별 폐업률로 깔때기를 분해하고, 몬테카를로로 **실행 가능한 배분안**까지 산출 |

> **포지셔닝:** Ferreira & Pereira는 "언제 들어가고 나올 것인가"를 이론적으로, CB Insights는 "얼마나 살아남는가"를 통계적으로 다뤘습니다. 본 프로젝트는 이를 **실제 데이터에서 결합** — Spark 집계로 산업·국가별 위험을 측정하고, 분산 시뮬레이션으로 포트폴리오 위험관리를 더해 투자자가 바로 쓸 수 있는 자본 배분안을 도출합니다.

### 에러요인 차단을 위한 전처리 방식 적용
`funding_total_usd`는 문자열에 통화 기호·콤마가 섞여 있어, **① 숫자/소수점만 추출 → ② 정규식으로 완전한 숫자 형태인지 재검증 → ③ 유효할 때만 `Double` 캐스팅, 아니면 `Null` 처리 → ④ 결측 행 제거** 순으로 정제했습니다. Spark의 ANSI 모드를 비활성화하여 형변환 실패를 안전하게 처리했습니다.

### 왜곡 보정 적용
투자금은 소수 대형 거래로 분포가 극단적으로 치우쳐 있기 때문에 평균이 왜곡됩니다. 이를 막기 위해 **① 상관분석에는 로그 변환**, **② 카테고리별 필요 자본에는 평균 대신 중앙값** 을 적용해 분석의 신뢰도를 높였습니다.

## 6. 분석 코드 스니펫

### Spark 분산 전처리 — 안전한 형변환
```python
import pyspark.sql.functions as F
from pyspark.sql.types import DoubleType

cleaned = F.regexp_replace(F.col("funding_total_usd"), "[^0-9.]", "")
is_valid = cleaned.rlike(r"^\d+(\.\d+)?$")
df_clean = (df.withColumn("funding_total_usd",
                F.when(is_valid, cleaned.cast(DoubleType())).otherwise(F.lit(None)))
              .dropna(subset=["funding_total_usd"]))
```

### 몬테카를로 포트폴리오 시뮬레이션
```python
np.random.seed(42)                          # 재현성 고정
for _ in range(num_portfolios):             # 10만 회
    weights = np.random.random(len(categories)); weights /= weights.sum()
    expected_success = np.sum(weights * success_rates) * 100
    funded = np.sum((total_budget_m * weights) / median_costs_m)   # 중앙값(롱테일 보정) 사용
    risk = (100 - expected_success) + max(0, (20 - funded) * 2)
```

### 핵심 결과 요약

|:---:|:---:|:---:|
| **0.27** (생존과는 0.06) | **29.8%** (최하위 1.3%) | 성공률 **78.7%** · 리스크 21.4 |

> 투자금 상위 구간 성공률 **1.3% → 29.8% (약 23배)** — 돈은 스타트업을 '생존'시키기보다 '성공 엑시트(인수·IPO)'시키는 것과 연결된다.

### 결론
투자금은 스타트업을 *살아남게* 하기보다 *크게 성공(인수·IPO)시키는 것*과 연결되므로, 자본을 다수 기업에 분산하기보다 **데이터로 성공률이 검증된 소수 산업군에 집중**하는 편이 리스크 방어와 기대수익 측면에서 합리적입니다.

### 제언 및 한계
본 결론은 **공개 데이터만으로 도출**되었기 때문에 다음의 제약을 함께 안고 있으며, 실제 투자 의사결정에 적용하려면 보완이 필요합니다.

- 밸류에이션·지분 희석·투자 시점 정보를 제공하는 플랫폼을 찾을 수 없어, 성과를 IRR이 아니라 폐업/성공-엑시트로 단순화하여 분석을 진행했습니다. → 실제 수익률·진입/회수 타이밍 관점의 결론은 앱개발 시장의 투자시점 및 회수 데이터 보강 후 재검증이 필요합니다.
- 정보의 부족으로 빈약한 조사가 진행되어 이를 보완하기 위해 `category_list`를 멀티태그로 설정한 이유로 바이오테크 성공률이 소수 표본(`Biotechnology|Health Care`, n=110)에서 89%로 과대평가되었고(단독 `Biotechnology` n=3,432는 16.1%), 이 값이 시뮬레이션에 그대로 들어가 "Biotech 집중" 결론이 과장되었습니다. → 따라서 실제 투자에 적용하기 전, 첫 태그 정규화 + 최소 표본 임계치(N≥200)로 성공률을 재산출해야 하며, 그 전까지 Biotech 단독 집중 투자의 결론을 위 분석의 최종 결론으로 인식해서는 안 됩니다.
- 리스크 점수가 휴리스틱 정의이고 NoSQL·실시간 처리를 제외했기 때문에, 향후 Markowitz 분산을 기반으로 한 위험모형과 생존분석·스트리밍을 더하면 자료 부족으로 포기했던, 과제의 주제인 투자 타이밍 분석까지 확장할 수 있을 것으로 기대합니다.
